# 個人レッスン予約管理システム

ともきコーチの個人レッスン予約を管理するWebアプリケーションです。

## 🎯 主な機能

### 顧客向け機能
- **2つの予約検索パターン**
  - パターンA: 施設から選んで空き日時を確認
  - パターンB: 希望日時から選んで移動可能な施設を表示
- **自動料金計算**
  - 社会人・ジュニア（中学生以下）の区分
  - 個人レッスン・グループレッスン対応
  - 1〜4名までの参加人数に対応
  - 1時間〜3時間までのレッスン時間（30分刻み）
- **柔軟な施設予約担当選択**
  - 顧客自身で予約
  - コーチに予約依頼

### 管理者向け機能
- **ダッシュボード**
  - 新着申請数
  - 今月の承認数
  - 今月の売上統計
- **予約申請管理**
  - 申請一覧表示
  - 承認・却下処理
  - 自動LINE通知メッセージ生成（コピペ用）
  - 施設予約担当がコーチの場合、電話番号・予約URLを表示
- **予定管理**
  - プライベート予定の追加（ラボ出稽古、家族、自主練など）
  - 予定一覧表示
  - 顧客側カレンダーに自動反映（詳細は非表示）
- **施設マスタ管理**
  - 施設の追加・編集・削除
  - 住所、電話番号、予約URL、予約方法の管理

## 🚀 デプロイ方法

### 1. Supabaseのセットアップ

1. [Supabase](https://supabase.com) でプロジェクトを作成
2. SQL Editorで `supabase-schema.sql` を実行してテーブルを作成
3. Authentication > Users で管理者ユーザーを作成
   - **重要**: "Auto Confirm User" にチェックを入れること

### 2. Vercelへのデプロイ

1. このリポジトリをGitHubにプッシュ
2. [Vercel](https://vercel.com) でアカウント作成
3. GitHubリポジトリと連携してデプロイ
4. 環境変数は不要（APIキーはHTMLファイルに直接記述済み）

### 3. アクセス

- **顧客向けページ**: `https://your-app.vercel.app/`
- **予約フォーム**: `https://your-app.vercel.app/booking.html`
- **管理画面**: `https://your-app.vercel.app/admin.html`

## 📊 データベース構造

### テーブル

1. **facilities** - 施設マスタ
   - 施設名、住所、電話番号、予約URL、予約方法

2. **lesson_requests** - レッスン予約申請
   - 顧客名、区分、レッスン種別、施設、日時、人数、料金、ステータス

3. **blocked_times** - ブロック済み時間
   - レッスン予定・プライベート予定など

## 📝 使い方

### 顧客側

1. トップページから「レッスンを予約する」をクリック
2. 検索方法を選択（施設から / 日時から）
3. 必要事項を入力して申請送信
4. LINEで承認連絡を待つ

### 管理者側

1. 管理画面にログイン
2. 新着申請を確認
3. 承認ボタンをクリック
4. 生成されたLINEメッセージをコピーして送信
5. 施設予約が必要な場合は電話またはWebで予約
6. 施設予約完了ボタンをクリック

## 🔐 セキュリティ

- Row Level Security (RLS) によるデータ保護
- 認証済みユーザーのみ管理機能にアクセス可能
- 顧客側は予約申請のみ可能（閲覧・編集不可）
- ブロック済み時間の詳細は顧客側に非表示

## 🐛 トラブルシューティング

### ログインできない
- メールアドレスとパスワードを確認
- Supabaseで「Auto Confirm User」を有効にしているか確認

### 施設が表示されない
- 管理画面の「施設マスタ」タブで施設を登録

### 予約申請が表示されない
- ブラウザをリロード
- Supabaseのテーブルが正しく作成されているか確認
